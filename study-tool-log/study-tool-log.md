# 학습 도구 로그

- 학습에 도움이 되는 도구를 만들고, 기록합니다. 필요하면 코드를 남깁니다.

## **도구 이름**
makeStudyLog

## **도구 유형** (예: GPTs, gems, Claude Code(skill, command, agent, ...) Chrome Extension, CLI, 웹사이트 등)
Claude Code skill

## **해결하려는 문제**: 어떤 학습 상의 불편/문제를 해결하려 했는가?
스터디 로그를 작성하는 과정에서 발생하는 반복작업을 줄이고, AI 피드백을 받는 과정까지 하나의 흐름으로 만들고 싶었습니다.

## **어떻게 만들었는가**: 간단한 제작 과정
공식문서를 먼저 살펴보고, 클로드 코드를 이용하여 지침을 상세하게 설정한 뒤 스킬을 제작했습니다.

## **어떻게 도움이 되었는가**: 실제 사용 경험과 효과
문제를 풀고나서 스터디로그를 작성할 때 이제 어떻게 적어야하나 걱정부터 시작됐었습니다.
스킬을 만들고 나서는 문제를 푼뒤 자연스럽게 해당 스킬을 실행시키게 되어서 문제풀이 - 스터디로그 작성하기 가 자연스러운 하나의 흐름이 되었습니다.
스킬을 실행시키면 주어진 질문에 답변하고 AI 피드백을 확인하는 과정이 문제를 푸는과정과 묶여있는 느낌이 들어 마음의 부담이 줄어들었습니다.





## **도구 이름**
study

## **도구 유형** (예: GPTs, gems, Claude Code(skill, command, agent, ...) Chrome Extension, CLI, 웹사이트 등)
Claude Code skill

## **해결하려는 문제**: 어떤 학습 상의 불편/문제를 해결하려 했는가?
타입스크립트를 하나도 모르는 상태에서 문제를 풀다보니 처음에 문제를 한번에 풀어내는 경우가 적었습니다.
이에 따라 AI를 이용해서 자연스럽게 키워드를 얻고 해당 키워드의 자세한 내용을 얻는 과정이 발생하게되는데, 자세한 내용을 얻는 과정에서 좀더 양질의 정보를 얻고싶었습니다.

## **어떻게 만들었는가**: 간단한 제작 과정
클로드코드를 이용해서 만들었습니다. 위에서 만든 makeStudyLog 스킬을 참조해서 원하는 지침을 작성하고, 빠르게 제작할 수 있었습니다. 

## **어떻게 도움이 되었는가**: 실제 사용 경험과 효과
키워드에대한 자세한 정보를 얻기위해서 항상 일일이 프롬프트를 길게 작성하려고 노력했었는데,
단순히 "/study {키워드}" 만 작성해도 원하는 정보를 쉽게 얻을 수 있었습니다.
프롬프트를 잘 작성해보아야겠다는 생각을 머리에서 지울 수 있었고, 키워드에 대한 정보를 얻고싶으면 주어진 정보를 잘 읽어보겠다는 생각만 할 수 있게 되었습니다.
