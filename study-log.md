### 1. 첫 학습 전략 설계

- 어떤 전략으로 학습하기로 했는가?
    1. Problem-Driven
    “이걸 왜 배우는가?” : typescript-exercises 1번 문제를 풀기 위해 부족한 지식을 찾아보기

    2. i+1
    타입스크립트 인터페이스 작성법 찾아보기

    3. Feedback Loop
    typescript-exercises 1번 문제 시작 -> 실패 -> 부족한 지식 확인 및 학습

    4. Structural Thinking
    옵시디언이나 마인드맵을 그릴 수 있는 툴로 찾아본 개념들을 이어가며 정리

    5. Resourceful
    해당 지식을 빠르게 습득하기위해 궁금한 부분만 AI에게 질문하여 확실하게 얻어내기.
    이미 타입스크립트를 잘하는 크루에게 학습 방향을 물어보기.

- 그 전략을 선택한 이유는?
    같은 시간 안에 더 많은 양의 학습을 하고싶고, 학습 과정에서 동기부여가 되는 목표설정을 한다는 것이 마음에 들어서.

- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.
    1. 일단 문제를 읽어보며 내가 뭘 모르는지 체크해봄
    2. 뭘 모르는지도 알 수 없어서 AI에게 이 문제를 풀기위한 키워드를 제공해달라고 물어봄
    3. 해당 키워드를 구글에 검색해보고 개인 블로그에서 문제를 풀기위한 지식을 습득
    4. 해당 지식을 바탕으로 문제를 풀었음
    5. 노션에 문제를 풀기위해 진행한 싸이클에서 얻은 지식을 정리


### 2. 전략 평가

- 효과적이었던 것과 그 이유
    목표에 필요한 지식을 습득하다보니 집중도가 올라가고, 자세한 내용을 포함한 지식을 습득하지 않았을때의 불만족도가 내려감

- 비효과적이었던 것과 그 이유
    TDD 싸이클과 비교하면 레드 - 그린단계만 수행한 느낌이라, 목표를 이루면서 궁금함이 생긴 지식들도 체크하고 지나가는 과정이 필요할 것 같음

### 3. AI 피드백

- 자신의 학습 전략에 대해 AI에게 피드백을 요청하고, 유용했던 제안 1가지 이상 기록

딱 하나의 개선안: "Failure Journal"을 도입하세요
제가 제안하는 멘탈 모델은 **"실패 지점의 구조화된 반복"**입니다.
지금 빠져 있는 것은 실패와 교정 사이의 성찰 단계예요. 현재 흐름이 이렇다면:

문제 시도 → 실패 → 검색/AI로 해결 → 다음 문제

이렇게 바꾸세요:

문제 시도 → 실패 → 실패 지점을 한 줄로 기록 → 검색/AI로 해결 → 같은 유형의 실패가 반복되는지 주 1회 확인

구체적으로, 노션에 개념 정리 대신 이런 테이블 하나를 만드세요.
이렇게 모이면 패턴이 보입니다. "아, 나는 타입 가드에서 계속 막히는구나" — 이 인식이 생기는 순간, 그 부분만 집중적으로 훈련할 수 있어요. 이것이 deliberate practice의 핵심입니다. 전문가는 자기가 어디서 반복적으로 실패하는지를 정확히 알고 있는 사람이에요.
개념 정리에 쓰던 시간의 절반만 이쪽으로 돌리세요. 정리는 필요할 때 검색하면 되지만, 자신의 실패 패턴은 기록하지 않으면 사라집니다.
